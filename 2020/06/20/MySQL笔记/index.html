MySQL学习笔记整理

MySQL中的重要数据类型

  1. 整型：

    tinyint：1字节

    smallint：2字节

    mediumint：3字节

    int：4字节

    bigint：8字节

    int可以指定长度，但必须和zerofill unsigned结合使用，否则没有意义，例如：id int(8) zerofill unsigned

  2. 浮点型：

    float，double

    double(M, D)表示一共有M位，其中D位为小数。

  3. 定点型：

    DECIMAL和NUMERIC都可以表示小数

  4. 时间类型：

    date：日期

    time：时间

    timestamp：底层用毫秒表示，可以区分时区，可以表示1970-2038年左右

    datatime：直接存储日期时间，不区分时区，可以表示1000-9999年

    year：只能表示年

  5. char：定长，补足的会用0补全，默认长度为1

    varchar：变长，必须指定长度，即varchar(M)，最后一个字节存放字符串长度。支持全文检索。

    text：也是字符串，但用的很少。

  6. 其他类型：

    xxxbit

    xxxblob：二进制类型，可以存二维码、小头像等

    枚举类型

    集合类型

  7. 特殊值：null

    所有类型都可以是null

SQL规范

  1. mysql对SQL语句不区分大小写，SQL语句关键字尽量大写，但数据库工具会给你自动转成大写。
  2. 字符型和日期时间类型使用单引号
  3. #表示单行注释；--也是单行注释，但后面要一个空格；/* */多行注释
  4. 只能是A-Z，a-z，0-9和_这63个字符的组合，且命名不要过长

SQL

三种SQL语句：DDL，DML和DCL。

DDL

数据定义语言，即定义库、表等，如下：

操作 命令
查看所有数据库 show databases;
创建数据库 create database 数据库名
删除一个数据库 drop database 数据库名
指定使用某个数据库 use 数据库名：运行本句之后，之后所有的sql语句都针对的是该数据库
查看某个库的所有表 show tables;，需要提前使用use
show tables from 数据库名
创建表格 create table [数据库名.]表名(字段名1 数据类型, 字段名2 数据类型，... );
查看表结构 desc <表名>;
增加一列 alter table 表名 add 字段名 数据类型 [after 字段名]/[first];，其中，after表示在某列之后添加，first表示在第一列添加
删除一列 alter table 表名 drop 字段名;
修改列类型，位置等 alter table 表名 modiry 字段名 数据类型 [after 字段名]/[first]
修改列名 alter table 表名 change 旧字段名 新字段名 数据类型 [after 字段名]/[first]
修改表名 alter table 表名 rename to 新表名
删除整张表 drop table 表名，它会删除表以及表中所有数据

创建数据库，选择字符集时，以_ci结尾的表示大小写不敏感，_cs结尾的表示大小写敏感。

DML

数据操作语言，即增删改查

操作 命令
添加数据 insert into 表名 values(值列表);
insert into 表名(字段列表) values(值列表);
insert into 表名 values(值列表1),(值列表2)...;：表示插入多行
查看数据 查看所有：select * from 表名 [where 条件];
只查看某些字段：select 字段列表 from 表名 [where 条件];
设置别名：select 字段1 as 别名1, 字段2 as 别名2... from 表名 [where 条件];，别名只在结果中显示
修改数据 update 表名 set 字段名1=字段值1, 字段名2=字段值2 [where 条件];
删除数据 delete from 表名 [where 条件];

数据完整性

实体完整性(Entity Integrity)：例如，同一个表中，不能存在两条完全相同无法区分的记录

用户自定义完整性(User-defined Integrity)：例如，用户名唯一，密码不能为空等

关联查询

mysql支持的关联查询有如下几种：

  1. 内连接： inner join
  2. 外连接：
    • 左外连接：left join
    • 右外连接：right join

mysql不支持全连接查询。

一共有7种结果：

  1. \(A\)：左连接

  2. \(B\)：右连接

  3. \(A\cup B\)：mysql使用union实现全连接，它的形式是\((A)union(B)\)

  4. \(A\cap B\)：内连接

    如果不写关联条件，会出现笛卡儿积。

  5. \(A - B\)：左连接

  6. \(B - A\)：右连接

  7. \(A\cup B-A\cap B\)：也是union，但形式是\((A-B)union(B-A)\)

事务

表示一组操作，要么同时成功，要么全部失败，这种操作组合就构成了一个事务。

涉及到的语句如下：

  • 开启事务：start transaction
  • 提交事务：commit
  • 回滚事务：rollback

对于事务，mysql默认自动提交，即执行一句自动提交一句，一旦提交无法回滚。可通过set autocommit = false;语句开启手动提交。

两个commit之间的操作算一个事务。

事务的特性：ACID

  • 原子性：要么同时成功，要么同时失败。一组事务必须小到不能再小，否则就拆开。
  • 一致性：指数据的规则，在事务前后必须一致
  • 隔离性：简单说，就是事务的操作对其他事务不可见。
    • 为什么要隔离：保证事务是独立的，与java中的线程安全问题类似。
    • 表现出来的问题：
      • 脏读：一个事务读取了另一个事务还未提交的数据
      • 不可重复读：一个事务读取了另一个事务提交的数据，导致一个事务期间对同一个数据的前后两次读取结果不同
      • 幻读：一个事务读取了另一个事务新增或删除并提交的数据，导致一个事务期间记录数改变了
    • mysql的隔离级别：
      • READ-UNCOMMITTED：允许事务读取其他未提及的数据，脏读、不可重复读、幻读都会出现
      • READ-COMMITTED：只允许事务读取其他事务已经提交的数据，可以避免脏读，但无法避免不可重复读、幻读
      • REPEATABLE-READ：默认的级别，不会出现以上三种问题，相当于行锁
      • SERIALIZABLE：相当于表锁，性能非常低
  • 持久性：当事务提交完成后，其影响应该保留下来，不能撤销

权限和安全

权限验证分两个阶段：

  1. 是否能够连接

    mysql验证连接的要求：主机地址+用户名+密码

  2. 验证权限

    • 全局权限：所有库，所有表，所有字段。
    • 对象级别权限：库->表->字段，以及每一种操作单独设置权限
