R语言-主成分分析与因子分析

发表于 2020-05-07 分类于 R语言 阅读次数： Disqus：

《R语言实战》笔记

主成分分析和探索性因子分析是两种用来探索和简化多变量复杂关系的常用方法。

思维导图

基本概念

主成分分析(Principal Component Analysis, PCA)是一种数据降维技巧，它能将大量相关变量转化为一组很少的不相关变量，这些无关变量称为主成分。主成分是观测变量的线性组合。形成线性组合的权重都是通过最大化各主成分所解释的方差来获得，同时还要保证各主成分间不相关。

探索性因子分析(Exploratory Factor Analysis, EFA)则是一系列用来发现一组变量的潜在结构的方法。因子被当作观测变量的结构基础或“原因”，而不是它们的线性组合。在EFA模型中，因子之间相关是常见的，但并不是必需的。

一条经验法则：因子分析需要5~10倍于变量数的样本数。

分析步骤

主要有以下7步：

  1. 数据预处理：PCA和EFA都根据观测变量间的相关性来推导结果。用户可以输入原始数据矩阵或者相关系数矩阵到principal()和fa()函数中。若输入初始数据，相关系数矩阵将会被自动计算。在计算前请确保数据中没有缺失值。
  2. 选择因子模型：根据你是要用于降维还是发现潜在结构，选择不同的方法。如果选择EFA，你还需要选择一种估计因子模型的方法(如最大似然估计)。
  3. 判断要选择的主成分/因子数目。
  4. 选择主成分/因子。
  5. 旋转主成分/因子。
  6. 解释结果。
  7. 计算主成分或因子得分。

主成分分析PCA

PCA的目标是用一组较少的不相关变量代替大量相关变量，同时尽可能保留初始变量的信息，这些推导所得的变量称为主成分，它们是观测变量的线性组合。

注意：主成分是预测变量的线性组合，而不一定是从预测变量中选一些变量（之所以说是"不一定"，是因为如果系数全是0或1，那就是从预测变量中选择一些变量，但这种情况非常少）！

例如第一主成分为：

\[ \mathrm{PC}_{1}=a_{1} X_{1}+a_{2} X_{2}+\cdots+a_{k} X_{k} \]

它是\(k\)个观测变量的加权组合，对初始变量集的方差解释性最大。第二主成分也是初试变量的线性组合，对方差的解释性排第二，同时与第一主成分正交。后面每一个主成分都最大化它对方差的解释程度，同时与之前所有的主成分都正交。

理论上来说，你可以选取与变量数相同的主成分，但实用来说，我们都希望用最少的主成分来近似全变量集。

PCA示例

本例使用Harman23.cor数据集，它包含305个女孩的8个身体测量指标。本例中，数据集由变量的相关系数组成，而不是原始数据集。

是否对数据进行标准化：看了网上的一些文章，有些说要标准化，有些说不用标准化。看到一个比较有说服力的答案：PCA(主成分分析)所对应的数学理论是SVD(矩阵的奇异值分解），而奇异值分解本身是完全不需要对矩阵中的元素做标准化或者去中心化的。但是对于机器学习，通常会对矩阵（也就是数据）的每一列先进行标准化。这样做可以避免某些数值大的列被过分关照，以及有利于梯度下降法的收敛。目前个人总结来看，上述这些说要标准化的，他们都是在用协方差矩阵进行计算，但在psych包中，是不用标准化的，因为它使用的是相关系数矩阵，已经被标准化了，所以不用自己手动标准化。

究竟要不要标准化，待博主再去研究研究。

判断主成分个数

最常见的是基于特征值的方法来决定主成分个数。每个主成分都与相关系数矩阵的特征值相关联，第一主成分与最大的特征值相关联，第二主成分与第二大的特征值相关联，依此类推。

可从3个方面来判断主成分个数：

  • 特征值大于1：Kaiser-Harris准则建议保留特征值大于1的主成分，特征值小于1的成分所解释的方差比包含在单个变量中的方差更少。
  • Cattell碎石检验：绘制特征值与主成分数的图形。这类图形可以清晰地展示图形弯曲状况，在图形变化最大处之上的主成分都可保留。
  • 随机数据矩阵：进行模拟，依据与初始矩阵相同大小的随机数据矩阵来判断要提取的特征值。若基于真实数据的某个特征值大于一组随机数据矩阵相应的平均特征值，那么该主成分可以保留。该访法叫作平行分析。

Proportion Var行表示的是每个主成分对整个数据集的解释程度。例如，此处PC1解释了整个数据集58%的方差，PC2解释了22%d方差，两者一共解释了81%的方差(Cumulative Var)

理念上的东西都不容易构建，当提取了多个成分时，对它们进行旋转可使结果更具解释性。

主成分旋转

旋转是一系列将成分载荷阵变得更容易解释的数学方法，它们尽可能地对成分去噪。

旋转方法有两种：

  • 正交旋转：使选择的成分保持不相关；
    • 最流行的正交旋转是方差极大旋转，它试图对载荷阵的列进行去噪，使得每个成分只由一组有限的变量来解释(即载荷阵每列只有少数几个很大的载荷，其他都是很小的载荷)。
  • 斜交旋转：让它们变得相关。

各成分的方差解释度趋同，准确来说，此时应该称它们为成分而不是主成分，因为单个主成分方差最大化性质没有保留。

获取主成分得分

我们的最终目标是用一组较少的变量替换一组较多的相关变量，因此，你还需要获取每个观测(不是变量)在成分上的得分。

前面讲的都是怎么提取主成分，但是我们最终得出主成分后是要拿去计算的，比如拟合模形，但现在只是知道了主成分与个变量之间的关系，但还没有一个具体的值。知道了映射关系，其实你也可以自行算出来主成分的值，但是很麻烦，而函数principal()提供的计算得分这个功能就是用来计算这个值的。

如果输入的是原始数据，那么principal()将会把得分结果保存到返回值的scores变量中；如果传入的是相关系数矩阵，那principal()函数只能返回主成分与原始变量之间的得分权重矩阵，存储在weights变量中，然后你再用这个矩阵去求每条观测的得分。

探索性因子分析

EFA的目标是通过发掘隐藏在数据下的一组较少的、更为基本的无法观测的变量，来解释一组可观测变量的相关性。这些虚拟的、无法观测的变量称作因子。每个因子被认为可解释多个观测变量间共有的方差，因此准确来说，它们应该称作公共因子。

模型的形式如下（提示，请与PCA的公式比较）：

\[ X_{i}=a_{1} F_{1}+a_{2} F_{2}+\cdots+a_{p} F_{p}+U_{i} \]

其中：

  • \(X_i (i=1...k)\)是第\(i\)个可观测变量；
  • \(F_j (j=1...p)\)是公共因子，\(p < k\)；
  • \(U_i\)是\(X_i\)变量独有的部分(无法被公共因子解释)；
  • \(a_i\)可认为是每个因子对复合而成的可观测变量的贡献值。

斜交旋转

对于斜交旋转，因子分析会考虑三个矩阵:

  • 因子结构矩阵(factor structure matrix)；
  • 因子模式矩阵(factor pattern matrix，变量与因子间标准化的回归系数)；
  • 因子关联矩阵(factor intercorrelation matrix，因子间的相关系数)。

"promax"斜交需要安装GPArotation包。

因子得分

相比PCA，EFA并不那么关注计算因子得分。

fa()函数也默认计算得分，如果传入的是实际数据，则得分存储在返回对象的scores变量中；如果传入的是相关系数矩阵，则返回的是得分权重矩阵，存储在weights中。

与可精确计算的主成分得分不同，因子得分只是估计得到的。fa()使用的是回归方法计算得分。

其他与EFA相关的包

FactoMineR包不仅提供了PCA和EFA方法，还包含潜变量模型。它有许多此处我们并没考虑的参数选项，比如数值型变量和类别型变量的使用方法。

FAiR包使用遗传算法来估计因子分析模型，它增强了模型参数估计能力，能够处理不等式的约束条件。

GPArotation包提供了许多因子旋转方法。

nFactors包提供了用来判断因子数目的许多复杂方法。
