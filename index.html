HTML中的表单

其实就是form标签，它的子标签如下：

  1. input type=text：文本输入框，value设置默认显示内容

  2. input type=password：密码输入框，value设置默认显示内容

  3. input type=radio：单选框，name属性对其进行分组，checked="checked"表示默认选中

  4. input type=checkbox：复选框，name属性对其进行分组，checked="checked"表示默认选中

  5. input type=reset：重置按钮，value属性修改按钮上的文本

  6. input type=submit：提交按钮，value属性修改按钮上的文本

  7. input type=button：普通按钮，value属性修改按钮上的文本

  8. input type=file：文件上传域

  9. input type=hidden：隐藏域，当我们要发送某些信息，而这些信息不需要用户参与，就可以使用这个标签，它的值也会被提交

  10. select：下拉列表框

  11. option：下拉列表框中的选项selected="selected"设置默认选中

  12. textarea：多行文本输入框(起始标签和结束标签中的内容是默认值)

    rows属性设置显示几行

    cols属性设置每行的宽度

js中的事件

onload加载完成事件：页面加载完成之后，常用于做页面js代码初始化

onclick单击事件：常用于按钮的点击响应操作

onblur失去焦点事件：常用于输入框失去焦点后验证其输入内容是否合法

onchange内容发生改变事件：常用于下拉列表和输入框内容发生改变后操作

onsubmit表单提交事件：常用于表单提交前，验证所有表单项是否合法

事件的注册

分为两种：静态注册和动态注册

静态注册

通过html标签的事件属性直接赋予事件响应后的代码。

动态注册

先通过js代码得到标签的dom对象，然后通过dom对象.事件名=function(){}形式赋予事件响应后的代码。

document是JavaScript提供的一个文档对象，表示整个页面。

JQuery

$是JQuery的核心函数，地位相当于js的document对象：

  • 当传入参数为html字符串时，为自动为我们创建这个html标签对象
  • 当传入参数为选择器时：
    • $("#id属性值")：id选择器，根据id查询标签对象
    • $("标签名")：标签名选择器，根据指定的标签名查询标签对象
    • $(".class属性值")：类型选择器，可以根据class属性查询标签对象
  • 当传入参数为dom对象时，会将其转换为JQuery对象

JQuery对象是dom对象数组，所以，可以通过下标将JQuery对象转换为dom对象。

JQuery对象不能使用dom对象的属性和方法，反之亦然。

attr()：可以设置和获取属性，但不推荐操作checked，readOnly，selected和disabled等属性。它还可以操作自定以的属性。

prop()：可以设置和获取属性，但只推荐操作checked，readOnly，selected和disabled等属性，与attr()互补

浏览器内核解析完页面标签之后，JQuery代码$(function(){})就会马上执行，先于原生js代码window.onload=function(){}执行。因为，原生js代码，除了要等浏览器解析完页面，还要等页面标签内容显示后才执行，比如加载图片。且，JQuery代码有多少个执行多少次，而原生js代码只执行最后一次的赋值。

return false可以阻止JQuery事件的冒泡。

动态web工程

创建的动态web工程有如下结构：

  • src目录：存放自己编写的java源代码

    • WEB-INF目录：一个受服务器保护的目录，浏览器无法直接访问到该目录中的内容

      • lib目录：一般这个目录是自建的，IDEA不会自动生成。主要用于存放第三方的jar包。
      • index.jsp：主页文件

HttpServletRequest

每次只要有请求进入Tomcat服务器，Tomcat服务器就会把请求过来的Http协议信息解析好封装到Request对象中。然后传递到service方法中(doGet和doPost方法)供我们使用。

HttpServletResponse

HttpServletResponse类和HttpServletRequest类一样，每次请求进来，Tomcat服务器都会创建一个Response对象传给Servlet程序去使用。HttpServletRequest表示请求过来的信息，HttpServletResponse表示所有响应的信息。我们如果需要设置返回给客户端的信息，都可以通过HttpServletResponse对象来设置。

设置字符集：resp.setContentType("text/html; charset=UTF-8")，它将同时设置服务器和浏览器使用的字符集；

请求重定向

它指客户端访问服务器，但服务器告诉客户端，你去我给你的这个地址访问。

它的代码是：resp.sendRedirct("URL")

请求重定向与请求转发的区别

请求转发由服务器内部转发请求，客户端只请求了服务器一次，浏览器地址没有发生变化；而请求重定向则是将连接发给客户端，再由客户端去访问，客户端请求了两次，且浏览器地址发生了变化。

为了解决servlet回传html困难的问题。

每一个.jsp文件其实就是一个Servlet。当我们第一次访问jsp页面时，Tomcat会帮我们把jsp页面翻译为一个java源文件，并将其编译为.class字节码程序，其中的类继承自HttpJspBase，而这个类其实又继承自Servlet。这个生成的字节码程序也是通过输出流将html内容回传给客户端。

4大域对象

其中又有4个对象被称为jsp四大域对象:

域对象名 类型 作用域范围
pageContext PageContextImpl 当前jsp页面内有效
request HttpServletRequest 一次请求内有效
session HttpSession 一次会话范围内有效（从打开浏览器访问服务器，到关闭浏览器为止）

这4个域对象的使用建议是：能用小的，就不要用大的，这样可以减轻服务器负担。

out.write()与response.getWrite()

这9大对象中，还有两个控制输出的out和response，它俩有如下联系：

  1. 当jsp加载完成后，会执行out.flush()清空out流的全部缓存，但它并不是输出到页面，而是将这些缓存输出到response.getWriter()流的缓冲区的末尾；
  2. 执行完上述操作后，才执行response.getWriter()流的刷新，将内容输出到最终的页面。

所以，response的输出永远要先于out的输出（如果你不手动执行out.flush()的话）。

如果查看源代码的话，可以发现，out其实就是response.getWriter()的引用。

为了不让输出变得错乱，建议只使用out.print()来输出数据。至于为什么不是out.write()，是因为out.write()只适合用来输出字符串，输出其他类型的数据会出问题。

Cookie

Cookie保存在客户端！

Cookie是服务器通知客户端保存键值对的一种技术。

客户端有了Cookie之后，每次请求都发送给服务器。

浏览器和Session之间的关联

这里是为了解释，为什么创建的Session对象明明没有超时，但是关掉浏览器再打开，使用的又是新的Session。因为这里使用到了Cookie。

服务器每次创建Session会话的时候，都会创建一个Cookie，它的生命周期是Session，它表示关闭浏览器后，它将被销毁。这个Cookie的键叫作JSESSIONID，它的值就是这个新Session的ID。换句话说，服务器通过返回Cookie的方式告诉浏览器本次会话的Session ID，但这个Cookie在关闭浏览器后就会被销毁。

创建了Session之后，浏览器每次请求又都会以Cookie的形式将Session ID一并发送给服务器，然后服务器根据Cookie中的Session ID再去服务器内存中找之前创建的Session对象。这也就是说，你每次调用req.getSession()方法，它都会通过读取Cookie找到Session ID，然后去内存找Session对象，找到了就返回，找不到就创建新的。

表单重复提交解决办法

如下情况可能发生表单重复提交：

  1. 提交完表单，但是服务器通过请求跳转来进行页面间的跳转，那么此时如果用户按F5，就会再次提交上次已经提交的表单，造成表单重复提交。这种情况下，可以使用请求重定向解决：response.sendRedirect()
  2. 用户正常提交服务器，但是由于网络延迟等原因，用户以为提交失败，于是用户多点了几次提交按钮，这时也会造成表单重复提交。
  3. 用户正常提交，服务器也没有延迟，但提交之后，用户回退浏览器到提交前界面，重新提交，这时也会造成表单重复提交。

第1种情况可以使用请求重定向，后两种则可以使用验证码来解决。

那么这里就涉及到了一个Filter执行顺序的问题，加入写了多个Filter，那它们究竟是按什么顺序执行的呢？执行顺序就是各个Filter在web.xml中的配置顺序。

多个Filter执行的时候还有如下特点：

  1. 多个Filter对象和目标资源都在同一个线程中；
  2. 多个Filter执行时共用同一个Request对象；

ThreadLocal的作用：

  1. 可以解决多线程数据安全问题；

  2. 可以给当前线程关联一个数据，如果要关联多个数据，则需要创建多个ThreadLocal对象。可以理解成它是一个Map对象，它的key就是当前线程。
